# KTC Server Temperature Bot

Телеграм-бот на Node.js: мониторит датчик температуры по JSON API и шлёт алерты в чат.

## Логика

1. **Детект превышения** — опрос API с заданным интервалом; при температуре выше порога считаем превышение.
2. **Алерт** — при первом превышении отправляется сообщение в чат.
3. **Повтор раз в 4 часа** — пока температура не падает, повторный алерт каждые 4 часа.
4. **Нормализация** — когда температура падает ниже порога, отправляется сообщение о нормализации, затем бот молчит до следующего повышения.

## Установка

```bash
npm install
cp .env.example .env
# Отредактируйте .env: TELEGRAM_BOT_TOKEN, TELEGRAM_CHAT_ID, TEMP_API_URL, TEMP_THRESHOLD
```

## Команды в Telegram

- **`/temp`** — запросить текущую температуру с API (ответ приходит в тот чат, откуда написали; в группах может понадобиться `/temp@ИмяБота`).
- **`/start`** — краткая справка.

## Запуск

```bash
npm start
```

Для команд включён **polling** у Telegram API — запускайте **один** экземпляр бота, иначе возможен конфликт `409`.

## Переменные окружения (.env)

| Переменная | Описание |
|------------|----------|
| `TELEGRAM_BOT_TOKEN` | Токен бота от @BotFather |
| `TELEGRAM_CHAT_ID` | ID чата для алертов (личный или группы) |
| `TEMP_API_URL` | URL JSON API (GET), возвращающего температуру |
| `TEMP_THRESHOLD` | Порог в °C (алерт при превышении) |
| `POLL_INTERVAL_MS` | Интервал опроса API в мс (по умолчанию 60000) |
| `TEMP_JSON_PATH` | Путь к числу в JSON, например `temperature` или `data.temp` |
| `STATE_FILE` | Путь к файлу состояния (по умолчанию `bot-state.json` в папке проекта) |

## Состояние на диске

В `bot-state.json` сохраняются `state` (`NORMAL` / `ALERTING`) и `lastAlertAt` (время последнего алерта в мс). После перезапуска:

- при уже активном превышении **повторно не шлётся** первый алерт;
- таймер **повторного** алерта продолжается с учётом `lastAlertAt` (остаток до 4 часов, не сброс с нуля).

## Формат API

GET-запрос к `TEMP_API_URL` должен возвращать JSON с числом температуры, например:

```json
{ "temperature": 42.5 }
```

или

```json
{ "data": { "temp": 42.5 } }
```

Во втором случае укажите `TEMP_JSON_PATH=data.temp`.
