# KTC Server Temperature Bot (MAX)

Бот для мессенджера [**MAX**](https://max.ru) на Node.js: периодически опрашивает JSON API с температурой и шлёт алерты в выбранный чат, если значение выше порога.

- **Библиотека:** [`@maxhub/max-bot-api`](https://www.npmjs.com/package/@maxhub/max-bot-api)  
- **Документация MAX для разработчиков:** [dev.max.ru — JavaScript](https://dev.max.ru/docs/chatbots/bots-coding/library/js)

---

## Возможности

| Функция | Описание |
|--------|----------|
| Мониторинг | Опрос `TEMP_API_URL` каждые `POLL_INTERVAL_MS` мс |
| Первый алерт | При переходе температуры выше порога — сообщение в `MAX_ALERT_CHAT_ID` |
| Повтор | Пока температура не снизилась — напоминание **раз в 4 часа** |
| Нормализация | Ниже порога — сообщение «в норме», сброс, ожидание следующего превышения |
| Команды | `/temp` — текущая температура; `/start` — справка |
| Состояние | `bot-state.json` — после перезапуска нет дубля «первого» алерта, таймер повтора продолжается от `lastAlertAt` |
| Логи | В консоль: `[MAX API]` — исходящие `sendMessageToChat` и входящие `message_created` |

## Требования

- **Node.js** ≥ **18.18** (требование `@maxhub/max-bot-api`)
- Доступ бота к вашему HTTP API с температурой (сеть / VPN при необходимости)

## Установка

```bash
git clone <репозиторий>
cd ktc-srv-temp-bot
npm install
```

Скопируйте пример окружения и отредактируйте:

```bash
# Linux / macOS
cp .env.example .env

# Windows (PowerShell)
Copy-Item .env.example .env
```

Обязательно задайте в `.env`:

1. **`MAX_BOT_TOKEN`** — токен бота в кабинете MAX (карточка бота).  
2. **`MAX_ALERT_CHAT_ID`** — числовой **id чата в MAX**, куда слать алерты (не путать с Telegram).

Остальные параметры — URL датчика, порог, путь в JSON (см. таблицу ниже).

---

## Запуск

```bash
npm start
```

Разработка с автоперезапуском при изменении файлов:

```bash
npm run dev
```

Запускайте **один** процесс бота: используется long polling MAX, второй экземпляр может конфликтовать.

## Команды в чате MAX

| Команда | Действие |
|---------|----------|
| `/start` | Краткая справка |
| `/temp` | Запрос текущей температуры с API (ответ в том чате, откуда написали) |

Команды бота в профиле задаются через `setMyCommands` при старте (`temp`, `start`).

## Переменные окружения

| Переменная | Обязательно | По умолчанию | Описание |
|------------|-------------|----------------|----------|
| `MAX_BOT_TOKEN` или `BOT_TOKEN` | да | — | Токен бота MAX |
| `MAX_ALERT_CHAT_ID` | да | — | Числовой `chat_id` чата для алертов |
| `TEMP_API_URL` | нет | см. пример в `.env.example` | GET, JSON с температурой |
| `TEMP_THRESHOLD` | нет | `50` | Порог в °C (строгое сравнение: алерт при **>** порога) |
| `POLL_INTERVAL_MS` | нет | `60000` | Интервал опроса API, мс |
| `TEMP_JSON_PATH` | нет | `0.temperature` | Путь к числу во вложенном JSON через точку |
| `STATE_FILE` | нет | `./bot-state.json` | Файл состояния (абсолютный или относительный путь) |

Файл `.env` не коммитьте: в нём секреты. В репозитории только `.env.example`.

## Формат ответа API температуры

HTTP **GET** на `TEMP_API_URL` должен вернуть JSON, из которого по пути `TEMP_JSON_PATH` извлекается **число** (температура в °C).

Пример: массив датчиков, берём первый элемент:

```json
[
  {
    "name": "ATC_E2CB06",
    "temperature": 30.18
  }
]
```

Для этого: `TEMP_JSON_PATH=0.temperature`.

Вложенные поля: `TEMP_JSON_PATH=data.sensors.0.temp` и т.п.

## Файл состояния `bot-state.json`

Содержит:

- `state` — `NORMAL` или `ALERTING`
- `lastAlertAt` — время последнего отправленного алерта (мс, Unix)

После перезапуска при уже активном превышении **повторно не отправляется** текст «первого» превышения; следующий **повторный** алерт планируется с учётом оставшегося интервала до 4 часов от `lastAlertAt`.

Файл указан в `.gitignore` как `bot-state.json` (локальное состояние машины).

## Отладка

- В консоли ищите префикс **`[MAX API]`**: исходящие сообщения в чат и входящие сообщения пользователей.
- Ошибки опроса датчика пишутся в stderr (`Ошибка запроса к API температуры`).
- Ошибки `setMyCommands` при старте не останавливают бота (предупреждение в консоли).
